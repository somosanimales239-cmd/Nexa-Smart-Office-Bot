CACHE
Cuando se guarda una URL o API Key nueva, Nexa elimina inmediatamente el ping
y connection-map anteriores y deja la conexion pendiente de una prueba nueva.
Test connection y Sync now guardan solamente el contrato recibido de la key
actual.

RESPUESTAS AUTOMATICAS
La sincronizacion de una conversacion usa mark_read=0. El mensaje no desaparece
como leido durante el analisis. Para usar DeepSeek automaticamente deben estar
habilitados Knowledge + authorized AI fallback y External AI fallback.
